java io概述

流
定义：流就是程序和设备之间有用于数据传输的管道，这根管道管道上有很多按钮，不同的按钮可以实现不同的功能；

分类：
Java.io包中定义多个流类型来实现输入输出功能：
（1）按数据流的方向不同可以分为输入流和输出流；
（2）按照数据处理的单位可以分为字符流和字节流；
（3）按照功能不同可以分为节点流（原始流）和处理流（包裹流）；
注：处理流就是包裹在原始节点上的流相当于包裹在管道上的管道；

我们主要学习以下几种流：
（1）四大基本流
J2SDK所提供的所有流类型分别继承以下四种抽象流类型
inputstream（字节流和输入流）、outputstream（字节流和输出流）、reader（字符流和输入流）、writer（字符流和输出流）
通常我们使用的它们的子类。
（2）文件流
通过文件流我们可以以字符或字节为单位将文件的内容读取或将内容写入文件中
分为：
字节流：FileInputstream、FileOutputstream
字符流: FileReader、FileWriter
(3)缓冲流
缓冲流是带有缓冲区的输入输出流；
缓冲流本身为处理流，不能单独存在，必须依附于节点流；
缓冲流作用：
1.可以显著减少我们对IO访问的次数，保护我们的硬盘；
2缓冲流要套接在相应的节点流上，对读写的数据提供了缓冲的功能，提高了读写的效率，同时增加了一些新的方法；
分类：
BufferedReader、BufferedWriter、BufferedInputstream、BufferedOutputstream
（4）数据流
它是以一种与机器无关的方式，将Java基本类型数据和string类型数据从输入流读取或写入到输入流；
分为：DataInputsream、DataOutputstream
(5)转换流
分为InputstreamReader（将Inputstream转换为Reader）、OutputstreamWriter（将Outputstream转换为Writer）
转换流为处理流，不能单独存在，必须依附相应的节点流
（6）print流
分为printWriter、printstream
print流的print方法可以写入所有基本类型数据的字符串形式表示
print流在基本输出流的基础上提供了增强的功能，可以方便的输出各种类型数据的格式化表示形式；
print流重载了print和println方法用于各种类型数据的格式化输出；
所谓格式化输出指的是将一个数据用其字符串格式输出；
printWriter和printstream区别：
作为处理流使用时，printstream只能封装Outputstream类型的字节流，而printWriter可封装Outputstream类型的字节流，
也可封装Writer类型的字符流；
printWriter中的println方法具有更好的跨平台性；
（7）object流
分为ObjectInputstream、Objectoutputstream
Object流用于对象的序列化；
对象的序列化是指将一个对象直接转化为字节流，然后将字节流直接写入到本地硬盘或网络中；
如果想将某个类对象序列化，则该类必须实现Serializable接口（该接口无任何方法，这种类型的接口被称为标识接口）；
如果一个类对象实现了Serializable接口潜在含义告诉编译器该类对象允许被序列化，若程序中已经存在序列该对象的代码，编
译器会自动处理来完成该对象的序列化；
如果一个类对象未实现Serializable接口，程序中存在序列该对象的代码，编译器会报错；
在Java中被transient修饰的成员变量在对象序列化时无法被序列化；
注：
（1）以stream结尾的流是字节流；
（2）以字节为单位读取数据显示时不能读取汉字否则会造成乱码；因为汉字的两个字节均包含有效信息，使用
Inputstream的read方法只能读取汉字的最后一个字节，这样解码时会出现错误；可以读取字母，虽然字母由两个字节存储，但
只有最后一个字节包含有效信息；
（3）字节流可以完成对各种信息的复制如文本、声音、视频、图片等，而字符流只能完成文本信息的复制；因为字节流无需解码
而字符流需要解码；
（4）对于输出的缓冲流，写入的数据会先存入缓冲区，使用flush方法可以清空缓冲区数据，将缓冲区数据全部压入目的地。只
有输出流有flush方法；
（5）BufferedReader中的readline方法用于读取一行字符串，遇到回车符会终止读取，并将读取到的回车符自动丢弃；
如果只输入回车，readline返回的是” “(空字符串)而不是null；如果想输入回车符，则调用BufferedWriter中的newline方法
（6）print流提供的打印方法不会抛出异常；
（7）DataOutputstream中的write方法是将数据的二进制数据写入文件；

赫夫曼树概述

赫夫曼树，又称最优树，是一类带权路径长度最短的树；

专业术语
结点的权值（weight）：树节点间与连线相关的数（如频率等）；
节点路径长度：从根节点到该结点的路径上的连线数；
树路径长度：树的每个叶子节点的路径长度之和；
结点带权路径长度：结点路径长度与它的权值的乘积；
树的带权路径长度（weighted path length）：树的每个叶子节点的带权路径长度之和；
WPL越小，二叉树性能越优，赫夫曼树为最优二叉树，WPL最小；

赫夫曼算法
1.根据给定的n个权值{w1,w2,w3,…,Wn}构成n棵二叉树的集合F={T1,T2,…,Tn},其中每个二叉树Ti（1=< i<= n）只有一
个带权为Wi（1=< i<= n）的根节点，其左右子树均为空；
2.在F中选取两棵根节点权值最小的树作为左右子树构成一棵新的二叉树，该二叉树的权值为其左右子树权值之和；
3.在F中删除选取的两棵二叉树，同时将新的二叉树放入F中；
4.重复2和3的操作直至F中只剩下一棵树，该树即为赫夫曼树;

赫夫曼编码
赫夫曼编码：可以有效地压缩数据（通常可节省%20~%90的空间，具体压缩率取决于数据的特性）
要设计长短不等的编码，则必须是任一个字符的编码都不是另一个字符的编码的前缀，这种编码称做前缀编码；
可以利用二叉树来设计二进制的前缀编码，约定左子树表示字符’0’,右子树表示字符’1’；从根节点到叶子节点的路径上分支字
符组成的字符串作为该叶子节点字符的编码，该编码必为前缀编码；
设计总长最短的二进制前缀编码即为以n种字符出现的频率为权，设计一棵赫夫曼树的问题得到的二进制前缀编码称为赫夫曼编码
赫夫曼树的每个结点包括四部分：该结点的权值、双亲游标、左孩子游标、右孩子游标；

非线性结构树的概述

树
专业定义：有且只有一个称为根的结点；有若干个互不相交的子树，这些子树本身也为一棵树；
通俗定义：树由结点和边组成（边指的是指针域）；每个结点只有一个父节点，但可以有多个子节点（根节点除外，根节点无父节点）；
专业术语：
父节点、子节点、子孙、堂兄弟（父亲为兄弟，他们的孩子为堂兄弟）、祖先；
深度：从根节点到最底层节点的层数；根节点为第一层；
叶子节点：无子结点的结点；
非终端节点：实际上为非叶子节点（有子节点的结点）；
度：子结点的个数
树的度：子结点个数的最大值；
树分类：
一般树
任意一个子结点的个数不受限制；
二叉树
任何一个节点的个数最多两个，且子结点顺序不可更改（有序树）；
分类：
一般二叉树
满二叉树：每一层的节点数都是最大节点数的二叉树；
完全二叉树：如果只是删除了满二叉树最底层最右边的连续若干个结点（此时深度未发生变化），这样形成的二叉树为完全二叉树；
满二叉树是完全二叉树的一个特例；
森林
n个互不相交的树的集合；
树的存储
二叉树的存储
连续存储【必须转化为完全二叉树进行存储：内存是线性一维的；存储不可只存储有效节点，否则无法还原二叉树】
优点：查找某个结点的父节点和子节点（也包括判断有没有子节点）速度快；
缺点：耗用内存空间过大；
链式存储：
耗用内存少，浪费空间少；
一般树的存储
双亲表示法
求父节点方便，求子结点难；
孩子表示法
求子结点方便，求父节点难；
双亲孩子表示法
求子结点和父节点都方便；
二叉树表示法
将普通树转化为二叉树来存储；
具体实现方法：
设法保证任意一个结点的左指针域指向他的第一个孩子，他的右指针域指向他的第一个兄弟；只要满足此条件就可将一个普通树转化为二叉树；
一个普通树转化成二叉树一定无右子树；
森林的存储
将森林转化为二叉树进行存储；
二叉树操作：
先序遍历：
1.先访问根节点；
2.再先序遍历左子树；
3.再先序遍历右子树；
中序遍历：
1.先中序访问左子树；
2.再访问根节点；
3.再中序访问右子树；
后序遍历：
1.先后序访问左子树；
2.再后序访问右子树；
3.再访问根节点；
先序中序后序将非线性结构转化为线性结构输出；
已知两种遍历方式求原始二叉树：
1.单个遍历方式无法还原二叉树；
2.已知先序和后序无法还原二叉树；
3.通过先序和中序 或 中序和后序可还原原始二叉树；
例如
已知先序：ABCDEFGH 中序：BDCEAFHG则后序为DECBHGFA;
已知先序：ABDGHCEFI 中序：GDHBAECIF则后序为GHDBEIFCA;
已知先序：ABFDCLMN 中序：BFACDLNM则后序为FBCNMLDA;
已知先序：MFLAQGBE 中序：FALGQMEB则后序为AGQLFEBM;
已知中序：BDCEAFHG 后序：DECBHGFA则先序为ABCDEFGH;
树应用：
1.树为数据库中数据组织的一种重要形式；
2.操作系统子父进程的关系本身为一棵树；
3.面向对象语言中类的继承关系本身为一棵树；
4.赫夫曼树（如何存树使效率达到最高）；

线性结构队列概述

队列：一种可实现先进先出的存储结构；
分类：静态队列（一般用循环队列实现）、链式队列；
1.静态队列一般为什么必须为循环队列？
传统数组实现队列会造成空间浪费；
2.循环队列需要几个参数来确定？
两个参数front和rear；
front永远指向队列的第一个元素，rear永远指向最后一个有效元素的下一个位置；
3.front和rear这两个参数在不同场合的不同含义？
队列初始化：front和rear值都为0；
队列为空：front和rear值相等但不一定为0；
队列非空：front为第一个元素，rear为最后一个有效元素的下一个位置；
4.循环队列入队伪算法：
将值存入rear所代表的位置；rear=（rear+1）/数组长度；
5.循环队列出队伪算法：
front=（front+1）/数组长度；
6.如何判断循环队列是否为满？
要么多设置一个参数，该参数名为长度，但频繁更新较麻烦；
要么存n个元素，找n+1个空间，空间浪费率为1/n+1，n越大，浪费率越低；这种方式front与rear相邻时为满；此时（rear
+1）/数组长度=front；
