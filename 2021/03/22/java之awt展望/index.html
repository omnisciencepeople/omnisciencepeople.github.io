java之awt展望概述

GUI：
“Graphical User Interface”的缩写，意为：图形化用户界面。

组件（Component）：是图形用户界面的基本组成元素，凡是能以图形化方式显示在屏幕上并能与用户进行交互的对象均为组件
如菜单、按钮、标签、文本框、滚动条等；
分类:java.awt.Component、java.awt.Menucomponent
java.awt.Component是抽象类，它是java除菜单相关组件之外的所有AWT组件类的根父类，该类规定了GUI组件的基本特性如尺
寸、位置、颜色等，并实现了一个GUI组件所应具备的基本功能;

容器：组件通常不能单独显示出来，必须将其存放于一定的容器中才可以显示出来；
有一类特殊的组件是专门存放其他组件的，这类组件叫做容器。
java.awt.container是所有容器的父类，继承自java.awt.Component;
容器类本身也是一个组件，具有组件的全部性质，但反过来组件不一定为容器。
容器分布关系：

Container:
Window(默认BoderLayout布局管理器):Frame和dialog
Panel（默认FlowLayout布局管理器）：applet

Panel:

public Panel();使用默认布局管理器FlowLayout创建新面板
public Panel(LayoutManager layout);使用指定的布局管理器创建一个新面板

布局管理器:
容器对其中包含的组件的排列方式，包括组件的位置和大小设定称为容器的布局；
为使图形用户界面具有良好的平台无关性，java语言提供了布局管理器来管理容器的布局，而不直接设置组件的尺寸和位置，每
个容器都有一个默认的布局管理器，当容器需要对某个组件进行定位或判断其大小尺寸时，就会自动调用其对应的布局管理器。
在AWT中常见的布局管理器：
BorderLayout、FlowLayout、GridLayout
FlowLayout
1.是panel类默认的布局管理器；
2.对组件进行定位，行内从左到右，一行排满后换行；
3.不改变组件原有大小，按照组件原有大小显示组件，可设置组件间的间距、行距及对齐方向；
4.FlowLayout默认的排列方式为居中；
5.常用方法：
new FlowLayout(FlowLayout.RIGHT,20,40);//右对齐，组件之间水平间距为20个像素，垂直距离为40个像素；
new FlowLayout(FlowLayout.LEFT);//左对齐，水平和垂直距离默认为5个像素
new FlowLayout();//默认为居中对齐方式，水平和垂直间距为5个像素
BorderLayout
1.是Frame类默认布局管理器
2.将容器划分为 东(EAST)西(WEST)南(SOUTH)北(NORTH)中(CENTER)五个区域，组件只能被添加到指定的区域
3.如果不指定组件的加入部位，则默认加入到CENTER区
4.每个区域只能加入一个组件，如加入多个，则先前加入的被覆盖
5.BorderLayout允许 南北区域水平缩放、东西区域垂直缩放、中部可任意缩放
GridLayout
1.该种布局方式将空间划分为规则的矩形网格，每个单元格区域大小相等，组件被添加到每个单元格。先从左到右填满一行后再从上往下填充。
2.填充时以行数为准；
3.构造方法：
public GridLayout();//在单个行中创建一个每个组件的默认值为一列的网格布局。
public GridLayout(int rows,int cols);//创建具有指定行数和列数的网格布局。 布局中的所有组件都具有相同的大小。
public GridLayout(int rows,int cols,int hgap,int vgap);//创建具有指定行数和列数的网格布局。 布局中的所有组
件都具有相同的大小。 此外，水平和垂直间隙被设置为指定的值。 每个列之间放置水平间隙。 在每行之间放置垂直间隙。
rows - 行，值为零表示任意数量的行
cols - 值为零，表示任意数量的列
hgap - 水平间隙
vgap - 垂直间隙

布局管理器总结：
1.Frame为顶级窗口，Frame缺省布局管理器为BorderLayout;
2.Panel无法单独显示，必须添加到某个容器中;
3.Panel缺性布局管理器为FlowLayout
4.当将Panel添加到某个容器中时，Panel仍可以拥有自己的布局管理器;
5.使用布局管理器时，布局管理器负责各组件的位置和大小，因此用户无法在这种情况下设置组件的大小和位置属性，如果想要
自定义组件的属性，则应取消默认布局管理器，调用setLayout(null);

事件处理：
事件：用户对组件的一个操作；
事件源：能够产生事件的GUI组件对象；
事件处理方法：能够接受、解析、处理事件类对象，实现与用户交互功能的方法；
事件监听器：可以处理事件的类对象；
事件处理步骤：
假定事件为A
1.向事件源注册某种事件的事件监听器 addAListener(aa);
2.设计好可以处理这种事件的监听器
class 类名 implements AListener{
重写AListener接口中的方法
}
要想设计能够处理A事件的监听器，只需编写实现了AListener接口的类即可，因为AListener接口已经定义了可以处理A事件的方法;
事件处理相关概念：
默认情况下事件源不会自动产生事件，程序员需要做两件事：
1.告诉事件源产生哪种事件即向事件源注册某种事件的事件监听器；
2.设计好可以处理这种事件的事件监听器；
一旦完成了这两步操作，当用户对事件源进行操作时，事件源就会自动产生事件并将事件封装成一个事件对象传递给事件监听
器，当事件监听器接受到事件源发送的对象时，会自动调用相应的事件处理方法来处理事件；
时间按有哪些？
java.awt.event包含了所有的事件，常见的事件：
Action Event:激活组件时发生的事件（如单击）
Key Event：操作键盘时产生的事件；
Mouse Event：操作鼠标时发生的事件；
Window Event：操作窗口时发生的事件，如最大化、最小化；

内部类：
定义在A类内部但定义在A类所有属性方法之外的类B称为内部类；
B为A的内部类，A为B的外部类；
内部类访问原则：
内部类可以访问外部类所有成员；
外部类的方法和属性不可直接访问内部类成员但可以new出内部类对象；
内部类优点：
可以让一个类方便的访问另一个类的所有成员；
增加程序的安全性，有效避免了其他不相关类对该类的访问；
什么时间使用内部类？
如果一个类A要使用B类所有成员，且A类无需被除B类以外的类或成员访问，我们应将A类定义为B类的内部类；

匿名类：
是一种特殊的内部类；
创建匿名类的三种方式：
1.实现接口
假入A为接口名
格式：
new A（）
{
实现接口中的方法；
}
功能：生成了一个实现A接口的匿名类对象；
2.实现抽象类
假定A为抽象类
格式：
new A()
{
实现所有抽象方法；
添加自己的方法或属性；
}
功能：生成了实现A类所有抽象方法的匿名类对象；
3.继承父类
假定A为类名
格式：
new A()
{
重写A中方法的代码；
添加自己的属性和方法；
}
功能生成一个A类的子类对象，该匿名类继承了A所有的非private成员；
如果在一个方法内部定义了一个匿名类，则该匿名类可以访问外部类的所有成员；
也可以访问包裹该匿名类方法的所有final类型变量；
