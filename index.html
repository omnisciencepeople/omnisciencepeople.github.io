用布尔函数判断是否为素数

#include<stdio.h>
#include<stdbool.h>
bool Isprime(int val);
int main()
{
int val;
char ch;
do
{
printf(“请输入你要判断的数字:”);
scanf(“%d”,&val);
if(Isprime(val))
printf(“Yes\n”);
else
printf(“No\n”);
printf(“请问你是否还要继续（Y/N）:”);
scanf(“ %c”,&ch);
}while(ch==’Y’||ch==’y’);
return 0;
}
bool Isprime(int val)
{
for(int j=2;j<val;j++)
if(val%j==0)
break;
if(val==j)
return true;
else
return false;
}

函数变量之全局变量

#include<stdio.h>
int A,B,C;//定义了三个全局变量;必须定义在函数前;
int max();//函数声明
int main()
{
printf(“请输入三个整数(中间以空格隔开):”);
scanf(“%d %d %d”,&A,&B,&C);
printf(“max is %d\n”,max());//A,B,C为全局变量所以在调用max函数时用不到传参，max可以直接使用A,B,C;
return 0;
}
int max()
{
int m;
m=A>B?A:C;
if(C>m)
m=C;
return m;
}

变量的存储方式和生存期

变量的存储方式有两种：静态存储方式和动态存储方式；
静态存储方式：指程序在运行期间由系统分配固定的存储空间的方式；
动态存储方式：指程序在运行期间根据需要进行动态的分配存储空间的方式；
内存中供用户使用的存储空间分为三部分：
1.程序区
2.静态存储区
3.动态存储区
全局变量全部存放在静态存储区，程序执行完毕就释放，在程序执行过程中，它们占据着固定的存储单元而不是动态地进行分配和释放；
在动态存储区中存放以下数据：
1.函数形参
2.自动变量
3.函数调用时的现场保护和返回地址等
对以上数据在函数调用时分配存储空间，函数结束时就释放；在函数执行过程中，这种分配和释放是动态的；如果在一个程序中两次调用同一函数（此函数中定义了局部变量）在两次调用时分配给这些局部变量的存储空间的地址可能是不同的；
局部变量的存储类别：
1.自动变量（auto变量）
函数的形参和函数的局部变量，生命周期和函数相同，函数调用结束后就自动释放这些存储空间；
auto 可省略；
2.静态局部变量
用关键词 “static”指定该变量为静态局部变量
函数结束后该变量所占用的存储的单元不会释放从而该局部变量的值得以保存；
静态局部变量缺点：占内存（长期占用不释放，而不能像动态内存那样一个存储单元可以先后为多个变量使用，节约内存）；
降低了程序的可读性（当调用次数过多时往往弄不清静态局部变量的当前值）；
3.寄存器变量
存储在CPU的寄存器中；
为提高执行效率；
对寄存器的存取速度远高于对内存的存取速度；
全局变量的存储类别
1.在一个文件内扩展外部变量的作用域extern
2.将外部变量的作用域扩展至其他文件extern
3.将外部变量的作用域限制在本文件中static
static声明一个变量的作用：
1.对局部变量进行声明，把它分配在静态存储区，该变量在整个程序执行不释放，其所分配的存储空间始终存在直至程序结束，函数内有效；
2.对全局变量进行声明，则该变量作用域只限于本文件夹不能被其他文件引用；

函数递归之流程解析

#include<stdio.h>
int f(int);//函数声明可不加形参，但必须末尾有引号；
int main()
{
int i=f(3);
printf(“i=%d\n”,i);
return 0;
}
int f(int i)
{
if(i==1)
printf(“哈哈\n”);
else
i= f(i-1);
return i;
}
输出：哈哈
i=1
函数调用流程：
f(3)->i=f(2)->i=f(1)->f(1)输出 哈哈;
return 1->f(1)完成->i=1->return 1->f(2)完成->i=1->return 1->f(3)完成;
跳出f函数 输出1；

函数递归概述

定义：函数直接或间接调用自己；
函数为什么可以自己调用自己？
前提：函数的调用是通过栈实现的；
A.一个函数在运行期间去调用其它函数，在运行被调函数前需做三件事：
1.将所有实参、返回地址（下一条指令的地址）等信息传给被调函数保存；
2.为被调函数的局部变量分配内存（包括形参）；
3.将控制转移到被调函数的入口从而执行被调函数；
B.从被调函数转移到主调函数前系统也需做三件事：
1.保存被调函数返回的结果；
2.释放被调函数所占用的内存空间（系统收回被调用函数占用的内存空间的使用权限）；
3.依照被调函数所保存的返回地址，将控制转移到主调函数（从被调函数跳出）；
C.当多个函数相互调用时，按照“后调用先返回”的原则，函数间的信息传递和控制转移必须借助栈来实现。即整个程序运行所需的数据空间安排在一个栈中。每调用一个函数时，就在栈顶分配一个存储区进行压栈操作，每当一个函数退出时，就释放它的存储区即出栈。当前运行的函数永远在栈顶。
D.在计算机看来，函数A调用函数B 和 函数A调用函数A是没有区别的。（执行步骤相同）
注：
不能无限压栈和出栈，否则系统会崩溃；
递归必须满足两个条件：
1.递归必须有明确的终止条件；
2.该函数所处理的数据规模必须在递减；n->n-1->n-2->n-3->…->1
递归和循环的关系：
所有循环都可转化为递归；所有递归不一定能由循环实现；
递归和循环优缺点：
递归：
优点：易于理解；缺点：速度慢、存储空间大（浪费空间多）；
循环：
优点：速度快、存储空间小（浪费空间少）；缺点：不易理解；
递归应用：
树和森林以递归方式定义；
树和图的很多算法都是以递归实现；
很多数学公式都是以递归方式定义的，如斐波拉契数列；

函数概述

为什么需要函数：
1.函数可避免大量重复性代码；
2.有利于程序模块化；
函数定义1：
逻辑上，能完成特定独立功能的代码单元块；
物理上，能接收数据，能对接收数据进行处理，能将处理结果返回；
函数适用范围：
适用大量同类型问题；
函数功能越单一越好（可移植性越强）；
函数定义2：
函数返回值类型 函数名（函数形参）
{
函数执行体
}
定义2本质：
1.详细描述函数之所以能实现某个特定功能的具体实现方法；
2.return表达式：向主调函数返回函数值；为空则表示终止函数；
3.函数返回值类型称函数类型；如果函数返回值类型和return表达式中数据类型不同的话，以函数名前返回值类型为主；（函数类型决定返回值类型）
函数分类：
有参函数和无参函数；
有返回值函数和无返回值函数；
库函数和用户自定义函数；
普通函数和主函数；
值传递函数和地址传递函数；
函数注意问题：
一个程序有且只有一个主函数；
主函数可调用普通函数；
普通函数不可调用主函数；
普通函数可相互调用；
程序从主函数入主函数出；
函数必须先定义后使用；
函数声明：
1.告诉编译器即将出现的若干个字母代表函数；
2.告诉编译器即将出现的若干个字母代表函数具体情况；
3.其为一个语句必加分号；
4.对库函数的声明是通过#include<库函数所在文件名.h>来实现的；
函数形参和实参：
个数相同；位置对应；类型相互兼容；
函数重要性：
函数为C语言的基本单位，类为Java/C++/C#的基本单位；
常见的系统函数：
sqrt:求平方根；
abs：求整型数据的绝对值；
fabs：求浮点型数据绝对值；
变量分类：
1.按作用域：全局变量和局部变量；
2.按存储方式：静态局部变量，自动变量，寄存器变量,静态外部变量，外部变量；
注：在一个函数内部定义的变量或函数的形参称为局部变量；
在所有函数外部定义的变量称全局变量；
全局变量和局部变量命名冲突问题：
在一个函数内部，若定义一个局部变量，其名与全局变量相同，局部变量会屏蔽掉全局变量；
俗称“强龙不压地头蛇”。
例如：
#include<stdio.h>
int i=99;//定义一个全局变量；
void f(int i)//定义一个局部变量，名字和全局变量相同；
{
printf(“%d\n”,i);
}
int main()
{
f(8);//调用f函数；
return 0;
}
//输出8，而非99；

比较3个字符串找出其中最大者

#include<stdio.h>
#include<string.h>
int main()
{
char str[3][20];//定义一个二维数组str，大小为3行20列，每一行可容纳一个字符串（最多为20个字符）；
char a[20];//定义一个一维数组作为中间变量交换字符串；
for(int i=0;i<3;i++)
gets(str[i]);//为每一行赋值
if(strcmp(str[0],str[1])>0)
strcpy(a,str[0]);
else
strcpy(a,str[1]);
if(strcmp(str[2],a)>0)
strcpy(a,str[2]);
printf(“%s\n”,a);
return 0;
}
